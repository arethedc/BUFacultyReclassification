# Basic Operations

This guide lists the core day-to-day actions in the Faculty Reclassification System.

## BO-01 - Sign In and Account Setup

1. Open the login page.
2. Enter your email and password.
3. If first-time user, use the activation email to verify and set password.
4. If needed, use Forgot Password to reset credentials.

## BO-02 - Create and Save Reclassification Draft (Faculty)

1. Open `Reclassification`.
2. Complete entries in Sections 1 to 5.
3. Attach evidence files to each applicable entry.
4. Click `Save Draft` anytime to store progress.

## BO-03 - Submit Application (Faculty)

1. Review section summaries and required fields.
2. Open submitted summary for final checking.
3. Click `Final Submit`.
4. Wait for success confirmation.

## BO-04 - Respond to Returned Application (Faculty)

1. Open the returned reclassification.
2. Check reviewer comments (Action Required first).
3. Update entries/evidence based on comments.
4. Reply to comments where required.
5. Click `Resubmit`.

## BO-05 - Review Queue Processing (Dean/HR/VPAA)

1. Open your review queue.
2. Select an application.
3. Validate entries, evidence, and section summaries.
4. Add comments (Visible to faculty or Internal).
5. Mark comment type (Action Required or FYI).

## BO-06 - Return to Faculty (Reviewer)

1. Click `Return to Faculty`.
2. Enter decision remark/reason.
3. Confirm action.
4. System updates status and notifies faculty.

## BO-07 - Forward to Next Stage (Reviewer)

1. Ensure checks are complete (including required workflow checks).
2. Click `Forward` / next-stage action.
3. Enter decision remark.
4. Confirm action.

## BO-08 - Resolve Comment Threads (Reviewer)

1. Open comment panel or inline comments.
2. Review faculty replies.
3. Mark as `Resolved` when acceptable.
4. Reopen/follow-up only when additional correction is needed.

## BO-09 - Manage Periods (Admin/HR)

1. Open period management.
2. Create a new period.
3. Activate period when submission window opens.
4. End period when cycle is complete.

## BO-10 - Track Logs and History

1. Use Status Trail to view workflow transitions.
2. Use Revision Log to see returned-entry changes.
3. Use approved/history pages for finalized records.

## BO-11 - Common Safety Checks

- Do not forward when required checks are still blocking.
- Confirm reason text for return/forward actions.
- Ensure evidence is attached before final submission.
- Verify role queue before taking reviewer actions.
